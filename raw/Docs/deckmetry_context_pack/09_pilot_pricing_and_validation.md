# 09 — Pilot Pricing and Validation

## Important framing

The pilot should not be treated as free custom software. It should be treated as a paid validation of a new sales and ordering channel for Wehrung's.

The pilot has value because it can:
- Test homeowner lead generation.
- Test contractor adoption.
- Reduce quote/order friction.
- Validate BOM accuracy.
- Identify integration requirements.
- Create data for a broader supplier platform.

## Recommended pilot length

Suggested pilot duration:
- 60–90 days after initial launch.

Why:
- 30 days is too short to gather meaningful contractor and homeowner data.
- 60 days is enough for early usage patterns.
- 90 days is better for multiple contractors, leads, revisions, and internal feedback.

## Suggested pilot structure

### Phase 0 — Discovery and configuration
- Confirm product categories.
- Confirm pricing assumptions.
- Confirm contractor workflow.
- Confirm homeowner workflow.
- Confirm admin review requirements.
- Confirm Epicor limitations and opportunities.

### Phase 1 — MVP build/pilot setup
- Homeowner path.
- Contractor path.
- Admin review dashboard.
- BOM generation.
- Export/email notifications.
- Basic pricing logic.
- Status workflow.

### Phase 2 — Live pilot
- Launch with limited contractor group.
- Embed or link from Wehrung's site.
- Test homeowner lead flow.
- Test contractor order request flow.
- Collect feedback weekly/biweekly.
- Track usage/conversions.

### Phase 3 — Review and commercial decision
- Review pilot metrics.
- Identify integration needs.
- Define full version scope.
- Decide pricing model and rollout.

## Pilot fee strategy

A pilot validation fee could be structured as:

### Option A — Low-friction strategic pilot
- Lower upfront fee.
- Good if you want Wehrung's to say yes quickly.
- Risk: underpricing the value and setting weak precedent.

### Option B — Serious paid pilot
- Mid-level pilot fee.
- Signals this is a real product, not a favor.
- Better if directors and IT are involved.

### Option C — Custom enterprise pilot
- Higher pilot fee with deeper customization.
- Appropriate if Wehrung's wants exclusivity, custom integrations, or significant control over roadmap.

## Suggested pricing concept

For a serious pilot, consider a paid validation fee in the range of several thousand dollars to low five figures depending on scope.

A possible structure:
- Pilot setup fee: paid upfront.
- Monthly pilot access/support fee.
- Clear statement that full Epicor integration is not included unless scoped separately.
- Full commercial subscription or licensing agreement after pilot.

## Protecting IP

Important:
Deckmetry should retain software ownership and IP unless Wehrung's pays for a much larger exclusive/custom development agreement.

Pilot language should clarify:
- Wehrung's is participating as a pilot partner.
- Wehrung's may receive a white-labeled or Wehrung's-branded version.
- Deckmetry retains platform ownership.
- Integration/custom features can be separately scoped.
- Exclusivity, if any, must be paid separately.

## Validation metrics

Track:
- Number of homeowner sessions.
- Number of homeowner BOMs generated.
- Number of labor quote requests.
- Number of contractor accounts invited.
- Number of contractor projects created.
- Number of submitted requests.
- Number of confirmed orders.
- Average order value.
- Time saved compared to email workflow.
- Number of admin revisions per BOM.
- Product selection trends.
- Feedback from sales/admin team.
- Feedback from contractors.

## Pilot success criteria

A successful pilot should prove at least some of the following:
- Contractors find it faster than creating/sending takeoffs manually.
- Homeowners submit qualified leads.
- Wehrung's receives cleaner order requests.
- Sales team can review and confirm without friction.
- BOM accuracy is acceptable after admin review.
- There is enough value to justify a paid full version.
- Epicor integration requirements become clearer.
