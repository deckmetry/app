# 04 — User Flows

## Homeowner flow

### Goal
Help a homeowner understand deck material cost, create a basic BOM, and become a lead for Wehrung's and/or an approved contractor.

### Flow

1. User enters through Wehrung's website.
2. User clicks **Homeowner**.
3. User sees a simple intro:
   - “Plan your deck, explore material options, and receive a material list.”
   - Disclaimer that pricing is estimated and subject to review.

4. User enters project basics:
   - Name
   - Email
   - Phone
   - Address or ZIP code
   - Deck width
   - Deck projection/length
   - Deck height
   - Attached to house or freestanding
   - Stairs: none / 1–3 steps / 4–8 steps / 9+ steps / custom
   - Railing needed: yes/no
   - Open sides requiring railing

5. User selects decking:
   - Brand
   - Collection
   - Color
   - Board type: grooved/solid as needed
   - Board length assumptions

6. User selects railing:
   - Vinyl
   - Composite
   - Aluminum
   - Cable
   - Color
   - Baluster or infill style
   - Stair rail if stairs selected

7. User selects options:
   - Fascia
   - Skirting
   - Lighting
   - Post/beam wrap
   - Hidden fasteners
   - Picture frame/border option in future versions

8. System generates:
   - BOM
   - Estimated material cost
   - Notes/disclaimers
   - Visual summary of selections

9. User can:
   - Email BOM to themselves.
   - Request pro labor pricing.
   - Ask Wehrung's to connect them with a contractor.
   - Save project.

10. Wehrung's admin receives:
   - Lead info
   - Project specs
   - BOM
   - Estimated material cost
   - Request type

### Homeowner CTA language

Primary CTA:
- “Send My Material List”
- “Request Pro Labor Pricing”
- “Have Wehrung's Review This Project”

Secondary CTA:
- “Save This Deck”
- “Talk to a Deck Specialist”

## Contractor flow

### Goal
Help contractors generate takeoffs faster and place/review orders with Wehrung's.

### Flow

1. Contractor enters through Wehrung's website.
2. Contractor clicks **Contractor Portal**.
3. Contractor logs in.
4. Contractor dashboard shows:
   - New Project
   - Draft Projects
   - Requested Orders
   - Confirmed Orders
   - Scheduled Deliveries
   - Completed Projects

5. Contractor clicks **New Project**.
6. Contractor enters:
   - Project name
   - Client name optional
   - Jobsite address
   - Deck width
   - Deck length/projection
   - Deck height
   - Joist spacing: 12” O.C. default, optional 16” O.C.
   - Framing options
   - Stairs
   - Railing sides
   - Material selections
   - Delivery preference

7. System generates:
   - BOM
   - Contractor-specific price
   - Product notes
   - Delivery request fields
   - Save/submit option

8. Contractor chooses:
   - Save as Draft
   - Submit for Wehrung's Review
   - Request Delivery Date
   - Download/Email BOM

9. Project status becomes:
   - Draft if saved
   - Requested if submitted

10. Wehrung's reviews and moves status forward.

### Contractor dashboard statuses

- Draft
- Requested
- Received
- Confirmed
- Scheduled
- Complete

## Admin/sales portal flow

### Goal
Allow Wehrung's to control review, pricing, availability, delivery, and communication.

### Flow

1. Admin logs in.
2. Dashboard shows:
   - New homeowner leads
   - New contractor requests
   - Requests needing review
   - Orders awaiting delivery scheduling
   - Backordered or problematic items
   - Completed orders

3. Admin opens a request.
4. Admin reviews:
   - Contact info
   - Project details
   - BOM
   - Estimated price/contractor price
   - Requested delivery date
   - Notes

5. Admin can:
   - Edit quantities
   - Adjust substitutions
   - Confirm availability
   - Add comments
   - Change status
   - Export BOM
   - Email customer/contractor
   - Create internal order manually or through future integration

6. Admin sends confirmation:
   - Confirmed price
   - Delivery/pickup date
   - Notes
   - Any items requiring substitution or special order
