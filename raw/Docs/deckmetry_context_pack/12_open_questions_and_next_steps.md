# 12 — Open Questions and Next Steps

## Open questions for Wehrung's

### Business questions
- Who owns the pilot decision?
- Who from sales/admin will use the system daily?
- Which contractors should be included in the first pilot?
- Should the homeowner flow show retail pricing or estimated ranges?
- Should Wehrung's connect homeowners to approved contractors?
- Does Wehrung's want this branded as Wehrung's, Deckmetry, or co-branded?

### Product questions
- Which decking brands should be included first?
- Which railing brands should be included first?
- Which categories are mandatory for MVP?
- Which products are stocked vs special order?
- Which BOM assumptions are acceptable?
- What disclaimers does Wehrung's want?
- Should delivery request dates be open text or calendar-based?

### Pricing questions
- How is contractor-specific pricing currently managed?
- Are contractor pricing tiers standardized?
- Can pricing be exported from Epicor?
- Should homeowner pricing show exact dollars or ranges?
- How often does pricing change?

### Epicor/IT questions
- What Epicor version is used?
- Does Epicor API access exist?
- Can product catalogs be exported?
- Can pricing be exported?
- Can quotes/orders be imported?
- Are there current integrations we can learn from?
- What security requirements apply?

## Recommended next steps

### Step 1 — Prepare Wehrung's demo
Create a realistic prototype flow with:
- Homeowner path
- Contractor path
- Admin path
- Example BOM
- Example status workflow

### Step 2 — Define pilot scope
Write a simple pilot scope document:
- What is included
- What is not included
- Pilot duration
- Pilot fee
- Responsibilities
- Success metrics

### Step 3 — Gather product/pricing sample data
Request from Wehrung's:
- Product list/SKUs
- Pricing examples
- Contractor pricing sample
- Common deck packages
- Railing product list
- Delivery workflow details

### Step 4 — Select pilot users
Choose:
- 3–5 contractors
- 1–2 Wehrung's sales/admin users
- Homeowner traffic from website/button

### Step 5 — Launch controlled pilot
Do not launch to everyone immediately. Start controlled, watch behavior, adjust fast.

### Step 6 — Review results
After 60–90 days:
- Present metrics
- Identify fixes
- Decide full rollout
- Scope Epicor integration

## Risks to manage

### Risk: BOM accuracy concerns
Mitigation:
- Use disclaimers.
- Keep admin review.
- Start with simple deck types.
- Allow manual adjustment.

### Risk: Epicor becomes a blocker
Mitigation:
- Use exports and manual workflow first.
- Treat integration as Phase 2.

### Risk: Contractors resist account setup
Mitigation:
- Make first use simple.
- Offer contractor onboarding.
- Show time savings.

### Risk: Wehrung's wants custom ownership
Mitigation:
- Clarify IP ownership early.
- Charge separately for exclusivity/custom development.

### Risk: Scope creep
Mitigation:
- Define MVP clearly.
- Keep future features separate.
- Use pilot success metrics to prioritize.

## Final recommendation

Go into the Wehrung's meeting with a clear story:

1. This can generate more homeowner leads.
2. This can make contractors more loyal.
3. This can reduce internal manual work.
4. Wehrung's remains in control.
5. Epicor integration can be phased.
6. The pilot should be paid and measured.
7. Deckmetry should retain platform ownership.
